## Key Points

1. **CLI tool that exports AI coding agent session logs** (conversations, tool calls, decisions) to structured JSONL datasets on Hugging Face. Converts raw session files into shareable, analyzable data.

2. **First-class OpenClaw support.** Dedicated parser for `~/.openclaw/agents/` — reads session JSONL files, extracts message pairs, tool invocations, and agent metadata. Also supports Claude Code (`~/.claude/`), OpenAI Codex (`~/.codex/`), Gemini CLI (`~/.gemini/`), Kimi CLI, and more.

3. **Multi-layer PII redaction** before any data leaves the machine:
   - Regex patterns: emails, phone numbers, IPs, API keys, tokens, SSNs
   - Entropy analysis: detects high-entropy strings (likely secrets)
   - Path anonymization: replaces `/Users/username/` with `/Users/[REDACTED]/`
   - Email redaction: `user@domain.com` → `[EMAIL_REDACTED]`

4. **Mandatory 4-stage attestation gate.** Before publishing to HuggingFace, user MUST confirm:
   - (1) Data has been reviewed for PII
   - (2) No proprietary code included
   - (3) Consent for public sharing
   - (4) Compliance with HuggingFace terms
   Cannot be bypassed — attestation is required every upload.

5. **Agent-native design.** JSON output includes `next_steps` fields for AI agent consumption. Can be installed as a Claude Code skill via `dataclaw update-skill claude`. Designed to be called by agents, not just humans.

6. **Lightweight.** Single dependency (`huggingface_hub`). Python ≥3.10. Install: `pip install dataclaw`.

## Architecture

```
~/.openclaw/agents/*/sessions/*.jsonl  ←  Raw session logs
         ↓ dataclaw export
    dataclaw_export/                   ←  Local JSONL (redacted)
         ↓ dataclaw upload
    HuggingFace Dataset               ←  Public/private dataset
```

**Parsers:** Each agent platform has a dedicated parser:
- `openclaw` → reads `~/.openclaw/agents/{agent_id}/sessions/*.jsonl`
- `claude` → reads `~/.claude/projects/**/sessions/*.jsonl`
- `codex` → reads `~/.codex/sessions/*.jsonl`
- `gemini` → reads `~/.gemini/sessions/*.jsonl`
- `kimi` → reads `~/.kimi-cli/sessions/*.jsonl`

**Export format:** Each conversation turn becomes a JSONL record with:
- `session_id`, `agent_id`, `platform` (openclaw/claude/codex)
- `role` (user/assistant/system/tool)
- `content` (redacted text)
- `tool_calls` (name, args, result)
- `timestamp`, `model_used`

## Commands

```bash
# Install
pip install dataclaw

# List available agent sessions
dataclaw list

# Export sessions to local JSONL (with PII redaction)
dataclaw export --platform openclaw --output ./dataclaw_export/

# Export specific agent
dataclaw export --platform openclaw --agent main --output ./dataclaw_export/

# Preview without writing (dry run)
dataclaw export --platform openclaw --dry-run

# Upload to HuggingFace (triggers attestation gate)
dataclaw upload --dataset username/my-agent-data --input ./dataclaw_export/

# Install as Claude Code skill
dataclaw update-skill claude

# Show stats
dataclaw stats
```

## Takeaways

1. **Useful for Iris data analysis.** Could export Iris's OpenClaw sessions to analyze conversation patterns, common failures, and agent delegation effectiveness. The structured JSONL is perfect for feeding into data pipelines or LLM analysis.

2. **PII redaction is solid but verify.** Multi-layer approach is good, but Iris sessions contain Zuma business data (store names, employee names, financial figures) that regex won't catch. Would need manual review or custom redaction rules before any public upload.

3. **OpenClaw parser is first-class.** Direct support for our exact directory structure (`~/.openclaw/agents/`). No configuration needed — it auto-discovers sessions.

4. **Agent skill install is interesting.** `dataclaw update-skill claude` installs as an agent-callable skill. Could similarly integrate with Iris's skill system if we want automated session export.

5. **Low priority for now.** Iris's memory system (pgvector + embed_memory.py + extract_signals.py) already captures the important signals from sessions. DataClaw would be useful for bulk analysis or sharing training data, but not urgent for daily operations.

6. **Security consideration.** Even with PII redaction, uploading agent sessions that contain SQL queries, database schemas, and business logic to HuggingFace would require careful review. Recommend local-only export mode for internal analysis.
