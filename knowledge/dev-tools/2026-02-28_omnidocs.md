## Apa Ini

Python toolkit dengan single API (`.extract()`) untuk semua document AI tasks: layout detection, OCR, text extraction, table parsing, structured extraction (Pydantic schema), reading order. Swap models/backends tanpa ubah code.

```python
pip install omnidocs
```

---

## Supported Tasks

| Task | Output |
|------|--------|
| Text Extraction | PDF/image → Markdown atau HTML |
| Layout Analysis | Bounding boxes + labels (title, table, figure, dll) |
| OCR | Text blocks + koordinat pixel |
| Table Extraction | Cells/rows/columns → DataFrame atau HTML |
| Structured Extraction | Typed data → Pydantic schema instances |
| Reading Order | Logical reading sequence |

## Cara Pakai (VLM API — No GPU)

```python
from omnidocs.vlm import VLMAPIConfig
from omnidocs.tasks.text_extraction import VLMTextExtractor
from omnidocs.tasks.structured_extraction import VLMStructuredExtractor

# Text extraction — tinggal set OPENROUTER_API_KEY / GOOGLE_API_KEY
config = VLMAPIConfig(model="gemini/gemini-2.5-flash")
extractor = VLMTextExtractor(config=config)
result = extractor.extract("document.png", output_format="markdown")

# Structured extraction ke Pydantic
class Invoice(BaseModel):
    vendor: str
    total: float
    items: list[str]

extractor = VLMStructuredExtractor(config=config)
result = extractor.extract("invoice.png", schema=Invoice, prompt="Extract invoice details.")
print(result.data.vendor, result.data.total)
```

## Multi-Backend Support

```python
# PyTorch local GPU
from omnidocs.tasks.text_extraction.qwen import QwenTextPyTorchConfig
# VLLM high-throughput
from omnidocs.tasks.text_extraction.qwen import QwenTextVLLMConfig
# MLX Apple Silicon
from omnidocs.tasks.text_extraction.qwen import QwenTextMLXConfig
# Cloud API (provider-agnostic via LiteLLM)
from omnidocs.tasks.text_extraction.qwen import QwenTextAPIConfig
```

## Supported Models

**Text Extraction:** VLM API (any cloud), Qwen3-VL, MinerU VL, Nanonets OCR2, Granite Docling, DotsOCR  
**Layout Analysis:** DocLayoutYOLO (0.1s/page), RT-DETR, Qwen Layout  
**OCR:** Tesseract (100+ lang), EasyOCR (80+ lang), PaddleOCR (CJK optimized)  
**Table Extraction:** TableFormer → DataFrame/HTML
